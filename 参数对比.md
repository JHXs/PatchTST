# PatchTST vs ST_PatchTST 参数对比

## 架构配置（arch_config）参数对标

### PatchTST (PatchTST.py)
```python
arch_config = dict(
    # 通用参数
    n_layers=3,           # 编码器层数
    n_heads=4,            # attention头数
    d_model=16,           # 模型维度
    d_ff=128,             # 前馈网络维度
    attn_dropout=0.0,     # attention dropout
    dropout=0.2,          # 通用dropout
    patch_len=24,         # patch长度  
    stride=2,             # patch步长
    padding_patch=True,   # 是否padding patch
)
```

### ST_PatchTST (ST_PatchTST_model.py)
```python
# PatchTST骨干网络参数（扁平结构，与PatchTST一致）
arch_config = {
    'n_layers': 3,            # 编码器层数 ✓ 对标
    'n_heads': 4,             # attention头数 ✓ 对标  
    'd_model': 16,            # 模型维度 ✓ 对标
    'd_ff': 128,              # 前馈网络维度 ✓ 对标
    'dropout': 0.2,           # dropout比率 ✓ 对标
    'patch_len': 24,          # patch长度 ✓ 对标
    'stride': 2,              # patch步长 ✓ 对标
    'padding_patch': True,    # 是否padding patch ✓ 对标
}

# ST_PatchTST特有参数（通过arch_config合并传递）
st_config = {
    'num_stations': int,              # 站点数量
    'feat_size': int,                 # 每个站点的特征维度
    'center_station_idx': int,        # 中心站索引
    'neighbor_hidden_dim': int,       # 邻站隐藏维度（轻量投影）
    'neighbor_dropout': float,        # 邻站dropout
}
```

## 对标说明

✓ **已对标参数**：
- `n_layers`: 3 层编码器
- `n_heads`: 4 个attention头  
- `d_model`: 16 维模型
- `d_ff`: 128 维前馈网络
- `dropout`: 0.2 基础dropout
- `patch_len`: 24 patch长度
- `stride`: 2 patch步长
- `padding_patch`: True patch填充

⊕ **ST_PatchTST 增强项**：
- `num_stations`: 多站点数量（PatchTST不需要）
- `feat_size`: 每站特征维度（PatchTST浓缩为c_in）
- `center_station_idx`: 中心站位置（时空聚合核心）
- `neighbor_hidden_dim`: 邻站轻量投影维度
- `neighbor_dropout`: 邻站专用dropout

## 参数流向

```
arch_config（扁平结构）
    ├─ PatchTST 参数 → __init__ 中直接提取
    │  （n_layers, n_heads, d_model, d_ff, dropout,
    │     patch_len, stride, padding_patch）
    │
    └─ ST特有参数 → __init__ 中直接提取  
       （num_stations, feat_size, center_station_idx, 等）
```

## 建议

对于训练对比实验：
1. **保持PatchTST参数完全一致**（n_layers=3, n_heads=4, d_model=16, d_ff=128, dropout=0.2, patch_len=24, stride=2, padding_patch=True）
2. **调整ST特有参数**（neighbor_hidden_dim, neighbor_dropout）
3. **记录数据集信息**（num_stations, feat_size, fcst_history, fcst_horizon）

对标对比的关键要点：
- ✓ 两个模型使用完全相同的PatchTST骨干网络配置
- ✓ ST_PatchTST额外的时空增强模块不影响基础预测器
- ✓ 差异来自于时空聚合层（邻站投影+门控机制）

避免混淆的做法：
- 不要嵌套 arch_config 多层
- 分清 TSForecaster 级别参数和模型级别参数
- 确保patch参数一致，否则对比实验无效
